Claude and Codex

Use chat to ask hotel questions.

Claude or Codex can check Resly for you and return a short answer a manager can use.

Claude

Good for managers who want to ask normal questions: available rooms, booking changes, price checks, and daily summaries.

  • Example "What should I know before check-in starts?"
  • Safety Claude can read and preview. It cannot quietly publish a change.

Codex

Good for building and improving the workflows: pricing checks, reporting scripts, and safe approval steps.

  • Example "Build a weekend price preview for these room types."
  • Safety Codex still has to use the same approval rules as everyone else.

What chat can do

Useful questions, simple answers.

Check availability

"Do we have rooms for 3 people next weekend?"

Summarize today

"What arrivals, departures, and changes need attention?"

Review prices

"Are we nearly full on Saturday, and should we preview a rate change?"

Find changes

"What bookings changed since yesterday?"

Prepare alerts

"Tell me when a reservation changes and what changed."

Explain the answer

"Show which rooms, bookings, and rates you checked."

For the technical handoff

The manager sees chat. The team gets a real integration.

Under the hood, this demo includes a Codex plugin and a Claude package. That gives Resly something practical to test without asking managers to learn API language.

Example manager question:
resly --json availability quote --guests 3 --from 2026-07-05 --to 2026-07-07

Codex plugin: plugins/resly-open-api
Claude package: claude/resly-mcp/resly-open-api.mcpb
Shared safe layer: resly CLI
